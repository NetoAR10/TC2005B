Preguntas a responder

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de redes de computadoras interconectadas que permite la comunicación entre dispositivos en todo el mundo. La World Wide Web es un servicio en línea que se ejecuta en Internet y que permite acceder y visualizar información a través de documentos multimedia enlazados entre sí, utilizando tecnologías como HTTP, HTML y URL.

¿Cuáles son las partes de un URL?

Protocolo: Indica el protocolo de comunicación a utilizar, como HTTP, HTTPS, FTP, etc.
Dominio: Dirección del sitio web al que se está accediendo.
Puerto (opcional): Número de puerto utilizado para la comunicación. Por defecto, HTTP usa el puerto 80 y HTTPS usa el puerto 443.
Ruta: Especifica la ubicación específica del recurso en el servidor.
Parámetros (opcional): Información adicional que se pasa al servidor.
Fragmento (opcional): Sección específica dentro del recurso.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita datos del servidor.
HEAD: Solicita encabezados de respuesta del servidor, sin el cuerpo del mensaje.
POST: Envía datos al servidor para ser procesados.
PUT: Reemplaza todos los datos de un recurso existente en el servidor.
PATCH: Modifica parcialmente un recurso existente en el servidor.
DELETE: Elimina un recurso del servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método POST se utiliza para enviar datos sensibles como contraseñas a través de formularios HTML, ya que los datos se envían en el cuerpo de la solicitud, no en la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Normalmente se utiliza el método GET para solicitar la página y sus recursos asociados.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Un código de respuesta 200 significa "OK". Indica que la solicitud fue exitosa y el servidor ha devuelto correctamente el contenido solicitado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un error 404, ya que indica que el recurso solicitado no se pudo encontrar en el servidor.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un error 500, ya que indica un error interno del servidor que impide que se complete la solicitud del cliente.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo HTML5 desaprobado significa que su uso se desaconseja y puede ser eliminado en futuras versiones de HTML. Algunos elementos de HTML 4 desaprobados en HTML5 incluyen "font", "center", y "strike".

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 introduce nuevos elementos semánticos como "header", "footer", "nav", etc.
Admite reproducción de audio y video nativa.
Introduce APIs para geolocalización, almacenamiento local, y canvas para gráficos.

¿Qué componentes de estructura y estilo tiene una tabla?

Estructura: "table", "tr", "td", "th".
Estilo: Puede aplicarse mediante CSS para cambiar la apariencia de la tabla, celdas, bordes, etc.

¿Cuáles son los principales controles de una forma HTML5?

"input" (text, password, checkbox, radio, etc.).
"textarea" para áreas de texto más grandes.
"select" para listas desplegables.
"button" para botones de envío u otros botones de acción.
"label" para etiquetar los controles y mejorar la accesibilidad.

¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

Google Chrome es conocido por ser uno de los navegadores que mejor soporta las especificaciones de HTML5.

¿Cuál es el ciclo de vida de los sistemas de información?

Identificación de necesidades: Se identifican y definen las necesidades del usuario y de la organización que el sistema de información debe abordar.
Análisis: Se analizan detalladamente los requisitos y se definen las funciones, características y restricciones del sistema.
Diseño: Se crea una arquitectura técnica y de software detallada para el sistema, incluidos los diagramas de flujo, las bases de datos y la interfaz de usuario.
Implementación: Se codifica, prueba e implementa el sistema de información en el entorno de producción.
Operación y mantenimiento: Se realiza el mantenimiento del sistema, se proporciona soporte continuo y se realizan actualizaciones y mejoras según sea necesario.
Retiro y reemplazo: En algún momento, el sistema puede ser retirado debido a la obsolescencia, la falta de soporte o la necesidad de una tecnología más avanzada.

¿Cuál es el ciclo de desarrollo de sistemas de información?

Planificación: Se define el alcance del proyecto, se establecen los objetivos y se elabora un plan de proyecto.
Análisis de requisitos: Se recopilan y documentan los requisitos del usuario y del sistema.
Diseño del sistema: Se diseña la arquitectura del sistema, la base de datos, la interfaz de usuario y otros componentes técnicos.
Desarrollo e implementación: Se codifica, prueba y despliega el sistema en un entorno de producción.
Pruebas: Se realizan pruebas exhaustivas para garantizar que el sistema cumpla con los requisitos y funcione correctamente.
Mantenimiento: Se proporciona soporte continuo, se corrigen errores y se realizan actualizaciones según sea necesario.
